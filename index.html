Canvas

Canvas is a element of HTML5. It's an amazing drawing technology built into all modern web browsers. With Canvas you can draw shapes, graphics, manipulate photos, videos, build games on the fly

History

  • was used by Apple in Safari and Dashboard in 2004
  • in 2005 in version 1.8 of Gecko browsers
  • in Opera in 2006
  • part of the HTML 5 specification
Firstly, I'll say some words about history. Canvas was initially introduced by Apple for use in their own Mac OS X WebKit component in 2004. Later, in 2005 it was adopted in version 1.8 of Gecko browsers, and Opera in 2006, and now it's a part of the HTML 5 specification .

How to use Canvas API

  1. Create the canvas element
  2. In JS, get your canvas element by using the id 
    const canvas = document.getElementById('myCanvas');
  3. Use the canvas element to get the context 
    const ctx = canvas.getContext('2d');
  4. Use the context to draw
Today I'm going to speach about Canvas API, which focuses on 2D graphics. Firstly, we should include canvas tag-element in HTML. Set id, width and height attribute. If a width or height is not specified, the default width of 300 pixels and the default height of 150 pixels are used. The "canvas" element is only a transparent container for graphics and has no border and no content. Canvas element requires the closing tag ("/canvas").If this tag is not present, the rest of the document wouldn't be displayed. The third step, we need to use getContext() method in javascript code. After that, ctx object has access to all of the drawing methods.

Coordinate space

Let's see the coordinate space. The canvas is a two-dimensional grid. Normally 1 unit in the grid corresponds to 1 pixel on the canvas. The coordinate (0, 0) is at the upper-left corner of the canvas. So the position of the top left corner of the blue square becomes x pixels from the left and y pixels from the top, at coordinate (x,y).

Properties: Color

fillStyle = color

strokeStyle = color

globalAlpha = transparencyValue

//create fill rect with opacity=0.5
ctx.fillStyle = 'rgb(255,0,0)';
ctx.globalAlpha = 0.1;

//create stroke rect with stroke opacity=0.5

For all shapes and path elements you can set color options. You can draw transparent elements in the path by using globalAlpha property. If you want set trancparent property only for one element, you should use alfa channel of rgb color.

Path

A path is a list of points, lines and shapes of different width and of different color are connected and form a shape.

If you want to draw more complex shapes, you should use paths.

How to use path

  1. Create the path
    beginPath()
  2. use drawing commands to draw into the path
    moveTo(), lineTo(), arc(), arcTo(), rect() and etc
  3. stroke or fill the path
    stroke() - draws the shape by stroking its outline
    fill() - draws a solid shape by filling the path's content area
  4. * closePath() - adds a straight line to the path, going to the start of the current sub-path
ClosePath method tries to close the shape by drawing a straight line from the current point to the start. It is an optional step, because when you call fill(), any open shapes are closed automatically,
moveTo(x, y)
Moves the pen to the coordinates specified by x and y

lineTo(x, y)
Draws a line from the current drawing position to the position specified by x and y

rect(x, y, width, height)
Draws a rectangle whose top-left corner is specified by (x, y) with the specified width and height
During second step of creating path you can use next methods: the moveTo() method is used to place the starting point to new position or to draw unconnected paths.

Clip

 
	ctxC.fillStyle = 'white';
	ctxC.fillRect(0,0,400,200);
	ctxC.fillStyle = 'red';
	ctxC.fillRect(0,0,400,100);
	ctxC.beginPath();
	//draw triangle
	ctxC.closePath();
	ctxC.lineWidth = 10;
	ctxC.stroke();
	ctxC.clip();
	ctxC.fillStyle = 'yellow';
	ctxC.fillRect(0,0,400,100);
Sometimes you may want to draw just part of a shape. You can do this with the clip function This can be useful for when you want to create a complex graphic by combining shapes, or when you want to update just a part of the screen for performance reasons.

Animation

  1. Clear the canvas
  2. Save the canvas state. Use save()
  3. Draw animated shapes, use requestAnimationFrame(callback)
  4. Restore the canvas state. Use restore()
If we need to move or transform shape in the canvas container we have to redraw it. If we want to save current settings before animation, we have to use save(). Draw new variant of the shape, you can use transformations. If we've saved the state, restore it before the next drawing.

Pixel manipulation

As I said before canvas gives you direct access to the pixels You can get the pixels for an entire canvas or just a portion, manipulate those pixels, For example, cnahge colors to gray scale and then set them back. These manipulations lets you do all kinds of interesting effects without Photoshop.

pros

  • perfect for dealing with many elements
  • pixel manipulations
  • has no DOM => realy fast => great for games
  • content can be updated at any time, new content can be drawn on old one
  • has a similar result on all platforms
This allows you to animate thousands of objects without unnecessary workload, which is ideal for games.

cons

  • slow when drawing to large areas(high resolution)
  • more complex than html&css
  • does not scale well
  • canvas content is not exposed to accessibility tools
  • text elements are invisible to search engines
In conclusion, I want to say than Canvas is a powerfull instrument for creating 2D and 3D elements. Which we can transform and animate in a different way.
